Zoning & Land Use Q&A App using RAG

This project enables intelligent zoning regulation queries based on geospatial coordinates. It combines spatial database querying, reverse geocoding, real-time zoning image retrieval, and a retrieval-augmented generation (RAG) system using LLMs.

  • Objectives:
    • Accept latitude/longitude input from users.
    • Identify zoning region and retrieve attributes.
    • Reverse geocode coordinates to addresses.
    • Display zone image dynamically.
    • Generate intelligent, regulatory-compliant responses.
    • Classify land use queries using RAG.
  • Technical Workflow:
    • Streamlit interface captures lat/lng.
    • PostgreSQL/PostGIS locates intersecting geometry.
    • Session-level memory stores metadata.
    • Nominatim/Mapbox for reverse geocoding.
    • Zoneomics API for zone image retrieval.
    • RAG setup with SentenceTransformer and LLaMA3.
    • Custom prompt generation for LLM.
